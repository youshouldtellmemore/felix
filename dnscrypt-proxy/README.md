# DNS Proxy Service

Raspberry Pi OS apt repository does not include dnscrypt-proxy, so it must be installed and upgraded manually.

# Install dnscrypt-proxy

Clone or copy this directory, then run [install-dnscrypt-proxy.sh](install-dnscrypt-proxy.sh).

```shell
chmod +x install-dnscrypt-proxy.sh
sudo ./install-dnscrypt-proxy.sh
```

The resolver is installed and configured to listen on 127.0.0.1:5053 to provide DoH to Cloudflare with DNSSEC enforcement.

The deployment is intentionally explicit:

- The dnscrypt-proxy built-in `-service install` behavior is not used.
- The long-running daemon runs as `dnscrypt-proxy:dnscrypt-proxy`.
- The root-owned binary and release assets live under `/opt/dnscrypt-proxy`.
- The config lives at `/etc/dnscrypt-proxy/dnscrypt-proxy.toml`.
- The service working directory is `/var/lib/dnscrypt-proxy`; resolver source caches live under `/var/cache/dnscrypt-proxy`.
- `/usr/local/bin/update-dnscrypt-proxy` is generated by the installer. It updates the root-owned release, validates the existing config with the new binary, and restarts through systemd.

Useful inspection commands:

```shell
systemctl cat dnscrypt-proxy
systemctl status dnscrypt-proxy
PID="$(systemctl show -p MainPID --value dnscrypt-proxy)"
readlink -f "/proc/${PID}/exe"
readlink -f "/proc/${PID}/cwd"
tr '\0' ' ' < "/proc/${PID}/cmdline"; echo
```
